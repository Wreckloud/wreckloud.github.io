Maven 简介

Maven 是 apache 基金会推出的一个开源的 项目管理工具, 它基于项目对象模型(project object model, POM)的概念.
Maven 通过一小段描述信息, 可以帮助我们 管理和构建 java 项目.

apache 软件基金会 , 是目前世界上最大且最受欢迎的开源软件基金会之一.
apache 所有的开源项目

Maven 的作用

  • 依赖管理: 自动下载项目依赖, 并管理这些依赖的版本.
  • 标准项目构建: 自动编译, 测试, 打包, 发布项目.
  • 统一项目结构: 项目结构的标准化, 使得项目更加清晰.

Maven 安装

  1. 下载并解压Maven:
    maven 官方提供的压缩包 apache-maven-X.X.X-bin.zip
    下载地址 .

  2. 配置仓库:

仓库用于存储资源, 管理各种jar包.

修改 Maven 下 conf/settings.xml 文件, 在 <localRepository> 标签下配置本地仓库路径.

我新建一个在 D:\Code\apache\ 下的 maven_repo 目录, 作为本地仓库.

那么我的 settings.xml 文件内容如下:

本地仓库：自己计算机上的一个目录.
中央仓库：由Maven团队维护的全球唯一的。仓库地址 .
远程仓库（私服）：一般由公司团队搭建的私有仓库.
本地仓库优先级高于远程仓库.

为了提升下载速度, 可以配置镜像仓库.
配置阿里云私服, 也在 conf/settings.xml 文件中配置 <mirrors> 标签下配置镜像仓库.

阿里云官方说明文档

  1. 配置环境变量:

为了能在任意目录下使用 Maven, 可以将 Maven 的 bin 目录添加到环境变量中.

在系统环境变量中添加 MAVEN_HOME, 值为解压后的 Maven 目录.
在用户环境变量中添加 PATH, 值为 %MAVEN_HOME%\bin.

在 IDEA 集成 Maven

配置 Maven 环境

我们并不会一直在命令行中使用 Maven, 绝大部分时候我们都会在IDEA 中集成 Maven.

可以针对当前项目配置 Maven , 不过那样每次新项目都要配置, 比较繁琐.
直接略过, 我们来配置全局 Maven 路径.

如果你的 IDEA 还在项目界面, 请先关闭项目返回开始界面.

然后在开始界面点击 All settings....

点击左侧 Build, Execution, Deployment -> Build Tools -> Maven.
配置 Maven 的全局路径等信息.
记得如果更改不了 User settings file , 请点击 override 按钮.

配置完成后点击 Apply 应用.
接着更改两个地方的 java 版本.


别忘记点击 Apply 应用.

创建 Maven 项目

新建项目

然后选择 Java 项目, 在Build tool中选择Maven, 然后点击下一步.

(注意: 不同版本的 IDEA 可能会有些许差异, 我这个是2024版本)

初次创建时, Maven 会自动下载依赖包, 请耐心等待.
